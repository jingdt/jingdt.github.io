知识管理国家标准解读 - 实施指南

发表于 2017-05-07

概述

从了解知识管理到实施知识管理是一个从理念、认知到实践落地的过程。知易行难，一个组织要想成功实施知识管理需要有充分的准备。实施过程可谓荆棘密布，到处充满绊脚石。为了提高实施 知识管理的成果率，应该对实施过程遵循的原则、方法和步骤有一个基本的了解。本文即是对这些通用原则的概要陈述，可以作为组织具体实施知识管理的参考。

实施前准备

识别变化因素

唯一不变的就是变化，知识管理实施过程涉及诸多不满、实施环节、业务人员和技术人员，都可能存在不可预知的变化因素。提前识别这些变化因素，可以针对性的制定应急预案、进行压力测试、实施场景模拟等措施，可以尽量降低实施过程带来的各种风险。

在进行业务交流、访谈、研讨的同时，知识管理实施责任人需要将庞杂的业务需求进行梳理和分析，要抓住核心关键需求，剔除大量漂浮的杂声。这时可以用业务需求分析评估工具进行评估，比如用需求性价比来分析。所谓需求性价比可以简化为一个公式：需求性价比 = 业务价值 / 工作量。具体来说，在做知识管理实施需求决策时，既要考虑需求本身的价值大小（业务价值、示范性、普适性），也要考虑需求实现的可行性（开发难度、业务管理成熟度、业务配合度）。

定义实施目标

知识管理实施的目标就是基于组织的发展战略和知识管理需求，提高组织整体协作水平，使知识管理的主体能够快速而方便地访问到所需要的知识。不断提高组织的创新能力、快速反应能力，提高组织效率和员工技能素质。目标和组织本身的特点紧密相关，有长期的也有短期的，有整体的也有局部的。可供参考的知识管理目标：

  • 在 100% 的时间内，在全球 100% 的地方，100% 地向客户提供完美的能力和资源。
  • 提高组织智商；
  • 减少重复劳动；
  • 避免组织失忆；
  • 发展并传递最佳实践；
  • 创造新产业；
  • 通过传播专有知识创造标准；
  • 知道你所知道的；

定义实施基准

在知识管理实施之前，有时候还需要定量地对组织机构当前的知识管理状态和水平做出评测，从而能够更好地定义知识管理实施基准。当实施知识管理之后，可以再次进行知识管理评估，则可以比较知识管理实施前后的差异和组织知识管理进步的水平。

知识管理成熟度模型（Knowledge Management Maturity Model，KMMM）是一个可以对企业或组织知识管理水平进行评估和分析的诊断模型，KMMM包括分析模型及发展模型。在“分析模型”中，将综合从战略、知识、流程、结构、技术、人员、文化、测量等领域对知识管理的各个方面进行透彻分析评测，同时还会重点对知识鉴别、知识创造、知识获取、知识存储、知识共享、知识应用等知识管理关键活动进行深入分析。其中“发展模型”则借鉴了软件能力成熟度模型（CMM），将知识管理的成熟度也分为 5 个层次：初始级、可重用级、可定义级、可管理级、可优化级。

通过 KMMM 分析模型可以精确揭示组织在知识管理方面潜在的可供提升之处，KMMM 的发展模型则可提供进阶下一级成熟度水平的有用的指导信息。将两个部分联合起来，可以确保知识管理的多个领域之间协调互联，同步发展。

KMMM的详细内容这里不展开，需要单独编撰文章进行阐述。

准备项目初步设计

如果项目预算审批通过，即可开始项目立项工作。项目立项后，就可以开始调用资源。这里需要确定项目实施是由组织内部自力更生为主还是引入外部第三方专业知识管理厂商为主。项目初步设计最重要的工作包括进度计划、成本计划、资源计划、采购计划、沟通计划等。

建立组织机构

开展知识管理的实施工作，要有可靠的组织保障。组织应根据自身情况建立专门的知识管理实施机构来确保实施工作的顺利开展。

确定实施阶段

当知识管理实施基准确定后，应遵照组织知识管理方向实施知识管理。根据组织的实际情况，将实施过程分为明确的几个阶段来进行推进。

实施阶段

实施准备工作就绪后，就进入具体实施阶段。一般来说，实施阶段包括 4 个阶段：认知、规划、试点、推广、运营。实施阶段可根据不同组织、不同行业、不同规模进行裁剪。知识管理在国家标准中提供了适用于大多数企业的实施过程阶段。

认知

知识管理其实质是一种组织变革。知识管理最为核心的难题是如何改变人的行为习惯和心态。因此，植入正确的理念，改变人的认知是实施知识管理的第一步，主要任务是明确知识管理对组织及个人发展的意义，统一组织全体人员对知识管理的认识。认知阶段的主要工作包括：

  • 对组织的员工（特别是中、高层）进行知识管理认知培训，全面完整地认识知识管理；
  • 利用知识管理成熟度模型等评估工具来客观认识知识管理现状并发觉自身的问题；
  • 组织文化和管理模式对知识管理采用何种实施方法有着决定性的作用，因此应特别注意不要忽略组织文化和管理现状。

规划

在规划阶段，首选要明确组织知识管理的导入模式是什么。导入模式不同其规划的着眼点和出发点也不一样。一般来说，组织知识管理导入的主要模式有如下几种：

  1. 战略导向 - 战略级知识规划：组织高层重视，自上而下的推动，他们对知识管理的关注点更多是强调知识/能力和企业战略的承接。
  2. 流程导向 - 流程级知识规划：业务部门重视或主导，或有业务推动能力的高层主导，他们在知识管理方面强调通过流程抓知识，再通过梳理知识来优化流程，使知识的沉淀、分享和流程执行过程紧密结合起来。
  3. 岗位导向 - 岗位级知识规划：业务部门重视或主导，或组织人力资源（HR）部门主导，他们在知识管理方面强调对核心岗位能力/知识模型的梳理，以及在此基础上的核心岗位知识沉淀和能力发展体系。
  4. 系统规划导向：IT部门重视或主导，从知识管理的 IT 系统规划、设计角度切入（尤其是企业知识门户规划的角度），他们在知识管理方面强调知识管理的系统功能规划及功能详细需求分析，以达到在内部满足业务部门需求、外部考察选择 KM 软件系统供应商的需求。
  5. 系统实施导向：IT部门重视或主导，强调知识管理系统在企业的实施推动。这种情况下，对KM厂商来说，只提供单纯的标准的软件平台和系统安装、配置服务已经难以满足企业的需求。因为企业实施知识管理是一个促成员工转变的过程，需要前期对配套系统的知识内容进行详尽的调研和建设，这些都是影响系统实施成败的关键。因此，知识管理系统应用实施咨询服务也成为了越来越多企业的选择。

知识管理规划主要包括以下几个方面的工作：

  1. 组织管理现状与知识管理发展成熟度分析。
  2. 从战略、业务流程及岗位等多层级进行知识梳理及知识规划。
  3. 明确知识管理的使命、愿景和战略目标，进行知识管理的蓝图规划，其中包括知识管理业务模式设计、知识管理组织结构设计、知识管理激励机制设计、知识管理规范指引设计、知识管理推广模式设计等。
  4. 进行知识管理 IT 系统规划，结合组织整体的 IT 系统规划以及技术路线，对涉及知识管理 IT 系统建设和发展进行规划，包括知识管理需求分析及优先级排序、知识管理 IT 建设路线、知识管理 IT 系统架构、门户及站点结构设计、分类体系设计等。
  5. 制定组织知识管理实施具体计划。

知识管理规划阶段的难点主要包括：知识管理和企业战略目标和业务流程的结合；知识管理与其他管理制度的结合及管理思想的转变；以知识管理思想为基础的业务流程的改造；知识管理的文化氛围的建立；知识管理规划与企业实际情况结合，建立适合企业自身特点的实践形式。

战略选择

知识管理战略选择主要包括两种：编码化策略和人际化策略。

  • 编码化策略是将知识与知识开发者剥离，使知识独立于特定的个体或组织，经过提取编码并存储于数据库中，以便随时查询提取使用。
  • 人际化策略是指知识与其开发者紧密地联结在一起，知识主要通过直接的面对面的接触来进行共享，更强调知识交流、协作与共享。

组织知识管理战略的选择，需要依据组织知识的特性进行选择。一般是将编码化策略和人际化策略结合起来考虑，根据企业自身特色有所侧重。企业的产品或服务是标准化还是个性化，是成熟的还是新颖的，企业知识类型是明确的知识还是难以言表的知识，直接影响知识管理战略的选择。

管理要素

在整个知识管理建设和实施过程中，必须同时关注其他重要的管理要素。主要包括：

  • 技术要素：一系列针对知识生产、存储、传递、共享的技术，是现代信息技术在知识经济时代的新发展。
  • 组织要素：主要是企业文化和人的观念变革。如知识私有观，害怕失去竞争力，不愿意分享自己的知识。
  • 流程要素：必须注意知识管理和业务流程的紧密结合。如果知识管理和业务流程脱节，将导致知识管理成为一项负担，不但没有促进员工工作效率，反而可能还会增加员工的不满情绪。

试点

试点是知识规划阶段的实践和延续，本阶段主要包括的工作：

  1. 根据不同业务体系的任务特性和知识应用特点进行知识管理模式分析，拟订合适的知识管理方案。
  2. 进行知识战略规划，识别关键知识，并判断关键知识的现状，进而在知识管理模式的指导下采取有针对性的提升行动。
  3. 按照知识管理里技术蓝图规划及食宿计划，完成知识库或知识管理系统建设。
  4. 结合组织业务模式进行知识体系梳理，引入知识管理系统对知识梳理结果进行有效管理，将整理的知识内容按照知识体系上传到系统平台上，并进行试点应用。
  5. 对知识管理平台应用情况进行全程跟踪、统计、分析和评价。
  6. 总结试点经验以确定知识管理具体模式和下一步推广计划。

推广和运营

在试点阶段不断修正知识管理规划的基础上，知识管理可以考虑在组织内全面推广，以全面实现其价值。本阶段主要包括以下几个方面的工作：

  1. 在组织中其他部门推广知识管理试点部门的实践经验；
  2. 知识管理全面地融入组织业务流程和价值链；
  3. 组织全面应用知识库或知识管理系统；
  4. 推广实践社区、学习型组织等知识管理实践活动；

评估与改进

知识管理实施进入日常运营后，需要不断进行阶段回顾和评审，以达到持续改进的目的。

建立评估改进机制

  1. 建立合适的评估改进组织
  2. 选择合适的评估指标
  3. 建立评估改进流程
  4. 落实持续改进的建议

知识审计

知识审计是指对组织当前的知识利用情况、知识管理的效果、整体氛围及环境等进行评估，并系统化地探究、分析、量度和评估组织知识资产的过程。

扩展知识管理

对知识管理投资回报率进行评估，主要可以参考三个评价标准：

  • 有效度：实施了知识管理后，整个业务过程的有效性提高了多少？
  • 扩散度：知识管理合理地实施了吗？项目和知识管理是否充分地理解了？
  • 编码度：知识编码化成本很高，组织应该限制吗？这种限制是否可见和被人理解？

制度化

组织知识管理体系的建设，需要长期坚持，这需要建立长效的管理保障机制，即制度化。

设置知识管理组织机构

常见组织模式：

  • 集中式
  • 分散式
  • 混合式

在组织模式基础上，还要设置对应的岗位。需分别从建设方项目团队和实施方向明团队职责进行考虑。

激励机制

激励问题在知识管理中是一个重要问题。激励机制更多地要知识量化，而知识量化是一个比较困难的事情。量化可以从数量和质量两方面考虑。知识量化及对知识型员工的激励机制是一个全球都在探索的核心问题之一。

总结

本文是继续对《知识管理良好实践指南》中“实施指南”章节的提纲摘录整理。由于知识管理国家标准针对的是企业而且是比较大型的企业，所以知识管理实施是一个大型项目的管理过程。涉及到的环节、内容、因素非常多，这部分只能以通用提纲形式表达大部分需要考虑的内容。在具体企业落地时，重点还是参考这里的指导性原则内容，而具体操作细节层面还是得结合每个企业自身的特点和个性化需求灵活进行规划和组织。
